# Releases

How security-atlas binaries are built, signed, and distributed.

Slice 039 wires the pipeline. This doc is the operator-facing reference.

## Distribution channels

| Channel               | Audience                 | URL pattern                                                     |
| --------------------- | ------------------------ | --------------------------------------------------------------- |
| GitHub Releases       | Anyone (raw download)    | `https://github.com/mgoodric/security-atlas/releases/tag/<TAG>` |
| Homebrew tap          | macOS + Linux brew users | `brew install mgoodric/tap/security-atlas-cli`                  |
| Install script (curl) | Linux + macOS scripted   | `curl -sSL https://get.security-atlas.io \| sh`                 |

## Supported targets (AC-1)

Each tagged release publishes archives for exactly these targets:

- `darwin_amd64` (macOS Intel)
- `darwin_arm64` (macOS Apple Silicon)
- `linux_amd64` (Linux x86_64)
- `linux_arm64` (Linux arm64 — Raspberry Pi 4+, Graviton, etc.)
- `windows_amd64` (Windows x86_64; zipped)

Each archive contains both `security-atlas` (server) and `security-atlas-cli`.
A second `security-atlas-cli_*` archive bundles the CLI alone (feeds the
Homebrew formula).

## Signing (AC-2)

The release pipeline cosign-signs the **checksums file** (which itself
covers every published artifact via SHA-256). We use Sigstore **keyless
OIDC signing** — there is no private key anywhere in the repo or in
GitHub secrets. Fulcio issues a short-lived (~10 min) certificate bound
to the workflow's OIDC identity. Rekor records the signing event in the
public transparency log.

### Verify a downloaded release

```sh
TAG=v0.1.0
VERSION="${TAG#v}"
BASE="https://github.com/mgoodric/security-atlas/releases/download/${TAG}"

curl -fSL "${BASE}/security-atlas_${VERSION}_checksums.txt"     -o checksums.txt
curl -fSL "${BASE}/security-atlas_${VERSION}_checksums.txt.sig" -o checksums.txt.sig
curl -fSL "${BASE}/security-atlas_${VERSION}_checksums.txt.pem" -o checksums.txt.pem

cosign verify-blob \
  --certificate-identity-regexp 'https://github.com/mgoodric/security-atlas/\.github/workflows/release\.yml@.*' \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  --certificate checksums.txt.pem \
  --signature  checksums.txt.sig \
  checksums.txt

# Then verify a binary against the now-trusted checksums.
sha256sum -c checksums.txt --ignore-missing
```

The release workflow runs the same verification step against its own
freshly-published assets (see `Self-verify signed checksums` step in
`.github/workflows/release.yml`) — if signing breaks, the workflow fails
loudly rather than shipping silently-broken artifacts.

## Homebrew tap (AC-3)

The formula is auto-generated by GoReleaser and pushed to a separate
tap repository:

```sh
# Tap once.
brew tap mgoodric/tap

# Install / upgrade.
brew install security-atlas-cli

# Verify.
security-atlas-cli --version
```

**Operator setup (one-time):**

1. Create the tap repo: `gh repo create mgoodric/homebrew-tap --public`
2. Issue a fine-scoped PAT with `contents:write` on that repo.
3. Add it as `HOMEBREW_TAP_TOKEN` in the security-atlas repo secrets.
4. The next tagged release will push the formula automatically.

If the PAT or tap repo is missing, `skip_upload: auto` in
`.goreleaser.yaml` keeps the rest of the release working.

## Install script (AC-4)

Hosted at `scripts/install.sh`. Production deployment of the redirect
`https://get.security-atlas.io` -> raw content URL is operator-owned
infrastructure (Cloudflare worker / static page). See
[`scripts/install.sh`](../scripts/install.sh) for full usage.

```sh
# Latest release into /usr/local/bin or ~/.local/bin.
curl -sSL https://get.security-atlas.io | sh

# Pin a version.
curl -sSL https://get.security-atlas.io | VERSION=v0.1.0 sh

# Cosign-verify in addition to sha256 (requires cosign on PATH).
curl -sSL https://get.security-atlas.io | VERIFY=cosign sh
```

The script always verifies SHA-256. Cosign verification is opt-in (we
don't want to silently require cosign on every install).

## Versioning

Semver. Tags are `vMAJOR.MINOR.PATCH`. Pre-release suffixes (`-rc.1`,
`-beta.2`) are honored — GoReleaser flags those as prereleases on GitHub.

## Release notes (AC-6)

Generated from Conventional-Commit messages between tags.
Groups (see `.goreleaser.yaml` → `changelog`):

- Features (`feat:`)
- Fixes (`fix:`)
- Performance (`perf:`)
- Refactors (`refactor:`)
- Documentation (`docs:`)
- Other work (anything else)

`chore: bump …` and merge commits are filtered out.

## Cutting a release (operator runbook)

1. Confirm `main` is green and the desired commits are merged.
2. Tag: `git tag -s v0.1.0 -m "release v0.1.0"` (signed annotated tag).
3. Push: `git push origin v0.1.0`.
4. Watch the **Release** workflow in Actions; it self-verifies on
   success.
5. If the tap repo or PAT isn't ready, the workflow still succeeds —
   only the brew push is skipped. Re-run after fixing.

**Do not retag.** Roll forward with the next patch.

## Local validation

```sh
just release-check        # validate .goreleaser.yaml
just release-snapshot     # build all 5 targets locally (no publish, no sign)
just install-script-test  # offline tests for scripts/install.sh
just print-version v0.1.0 # confirm ldflags wiring
```
