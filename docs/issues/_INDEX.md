## Index policy

This file is the **v1 spec snapshot** — the static record of the 58 originally-scoped v1 slices plus the in-flight additions (052–058) that landed in the v1 timeframe. It is **not** the live status tracker; the row-level `Status` column captures the historical "Ready for execution" state at backlog-freeze time, not current-on-`main` state.

The live merge tracker is [`_STATUS.md`](./_STATUS.md) — that file is the source of truth for what has merged, what is in-progress, what is in-review, and what new slices have been filed since v1 backlog-freeze.

**v1 expansions kept in the index:** the 11 v1 slices added during execution (052–058 plus 051) are listed here because they were scoped against the v1 success test (canvas §10.1) and tracked through the same backlog-review process.

**Post-v1 slices are NOT in this index by design:** slices 059 onward (feature flags, path-filter CI, admin endpoints, security audit remediation, logo work, infra fixes, follow-on work) are v2 / post-v1 follow-ons. They live in [`docs/issues/`](./) and in `_STATUS.md` drift sections, not here. Adding them would conflate "what v1 was supposed to be" with "what has happened since v1 completed" — two distinct questions the maintainer asks separately. The historical-record purpose of this index is the reason. New slices should be filed via the per-slice template (`Plans/prompts/04-per-slice-template.md`) and surfaced in `_STATUS.md`; do not edit this file to add them.
