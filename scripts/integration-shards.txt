# INVARIANTS this file encodes (slice 417 ACs / P0s):

#   * P0-1 / AC-6 — the UNION of all legs' package args MUST equal the
#     current integration-enrolled set EXACTLY (no package unassigned,
#     none double-assigned). scripts/check-integration-shard-coverage.sh
#     proves union == the ci.yml `go test` list and fails CI on drift.

#   * P0-2 / AC-3 — the SHARED-SEED cluster (the SCF global catalog
#     `scf_anchors` [no tenant_id] + the schema-registry catalog rows in
#     `evidence_kind_schemas`) stays in LEG A (serial, -p 1). Leg A is the
#     ONLY leg that seeds/imports the SCF catalog, so no two parallel
#     Phase B shards can race on `scf_anchors` cross-binary. The slice-461
#     order-independence guard + the migration round-trip both run on
#     Leg A, where the catalog lives.
#
#   * P0-4 / AC-4 — every leg (A, B1, B2, B3) runs `-p 1` INTERNALLY.
#     The parallelism is across runners (each Phase B shard is a matrix
#     leg with its OWN Postgres/MinIO/NATS), never `-p N` within a runner.
#

#   * Leg A  = shared global-catalog seeders + the eval/control consumers
#              tightly coupled to the freshly-seeded SCF catalog. (Slice 747
#              moved ./internal/api/controls/... — a ~228s self-seeding,

#   * Leg B4 = ./internal/api/controls/... (the ~228s monolith moved off A)
#              + the cross-cutting read handlers shed from B3 (mcp, llm,
#              freshness/drift/staleness, exception, observability, platform).
#              (Slice 747 NEW leg — a matrix leg, NOT a new required check.)
#   * Leg B5 = the other half of the OSCAL import cluster (oscal +
